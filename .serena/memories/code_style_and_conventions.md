# コードスタイルと規約

## フォルダ構成ルール
- `data/`: 安定データ格納
- `scripts/`: メインスクリプト
- `docs/`: ドキュメント
- `tmp/`: 検証用・一時データ

## コーディング規約
### テスト駆動開発
- t-wadaのTDDを参考にテストを書く
- 検証は実際のデータと比較すること

### ログ処理
- 標準loggingモジュールを使用
- loggerで統一してログ出力

### データ処理
- polarsを主要データフレームライブラリとして使用
- 大容量データはチャンク処理で分割
- pickleファイルでの中間保存を活用

### 開発プロセス
1. 検証段階でコードを書く
2. コードを修正
3. 結果を確認
の流れで進める

## 命名規則
- クラス: PascalCase (例: ScalableHierarchyScraper)
- 関数・変数: snake_case
- 定数: UPPER_CASE
- ファイル: snake_case.py
