# タスク完了時のワークフロー

## 基本的な開発サイクル
1. **検証段階**: 実際のデータを取得してコードの結果と比較
2. **コード修正**: 検証結果を基に改良
3. **結果確認**: 期待される出力が得られるかチェック

## データ品質確保
- スクレイピング結果は実際のWebページと照合
- テストケースによる自動検証実行
- 統計情報で異常値をチェック

## ログとデバッグ
- 全処理でloggerを使用した詳細ログ出力
- 実行時間とメモリ使用量の測定
- エラー時の詳細情報記録

## データ管理
- 安定版データは`data/`に格納
- 検証用・一時データは`tmp/`に格納
- pickleファイルでの中間データ保存

## 完了基準
- 実データとの比較検証でコンプライアンス100%
- 処理統計の妥当性確認
- ログによる処理過程の透明性確保

## 推奨する検証コマンド
```bash
# 総合的な結果検証
uv run python scripts/quick_tag_analysis.py

# 特定機能の検証
uv run python scripts/query_body_parts.py
```
